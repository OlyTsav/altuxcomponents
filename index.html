Introduction

How To Use This Guide

This guide acts as a reference for designers and developers for creating and implementing designs for Alt UI components. Throughout each section you will find a description and any applicable examples for usage. If the example contains a code sample, it may be copied for direct implementation using the Alt UI SDK, harnessing Angular.js v1.5.

Recommend A Style

If you notice a style is missing or you have created a new component with unused styles, use the “Recommend a Style” link in the top right of this guide. All submissions will be sent to the style owner and discussed with you prior to implementation into this guide.

For questions regarding implementation of anything you see here, contact justinay@microsoft.com

Code Samples

Code samples can be copied directly from this guide. They are dependent on the latest SDK which includes Angular.js v1.5. Code samples may need to have new inline Angular requests added or altered in order to work for your specific application. For recommendations of more efficient writing or structuring of code, email justinay@microsoft.com.

Motion

All animations use the same curve structure. Entrance and some exit animations use the same curve, but are reverse of each other. Some exit animations are quicker than others. This will be explained further in the guide

Animation Curve and Time

Each animation has a maximum time allocation of .5s. Using jQuery Easing, we call the easeOutExpo property which will match our animation curve.

Entrances

Used for page loading and adding elements that are requested via agent actions. Headers, sidebars, and main content all have different directions they enter from, see section

Ultimately all animations use their motion to form a hierarchy and better engage the user with their content. For now, all entrances have the same curve and timing applied to that curve. The curve is pulled directly from MWF Motion guidelines for elements entering view, and have a duration of .5s

First Time Entrances

The first time an element enters the screen it does so through the entrance animation. If the element is in a group of other similar ones, such as a tile, it is loaded in a linear order with a .15s delay between element start animation in the group.

Exits

Used for switching views, from one location in the dashboard to the other. Also used when an element on the page is being removed from view or replaced with new updated data. The two animations are different in that an action like replacing content will use the reverse of the entrance animation. But switching views will remove all the elements much quicker as to allow more time for the next entrance of the new screen.
